# LEX/Flex Practice

## Description
This project is a repository of practical exercises and scripts developed to master **LEX/Flex**, a tool for generating scanners (lexical analyzers). It contains various small programs that process text input to identify specific patterns, such as counting characters, identifying valid variable names in programming contexts, and echoing or transforming input strings.

## Technologies
-   **C11**
-   **Flex** (Fast Lexical Analyzer Generator)
-   **GCC** (GNU Compiler Collection)

## Features
-   **Symbol Counter**: Analyzes text to count the occurrence of specific symbols or characters.
-   **Line Tracker**: Counts and displays the number of lines in a provided source file.
-   **Variable Identifier**: Uses regular expressions to detect and validate variable names based on specific naming conventions.
-   **Echo and Transformation**: Basic scripts to demonstrate input/output handling and simple string manipulations within Flex.
-   **Exercise List**: A dedicated sub-folder containing a series of progressive challenges (ex1 through ex6) for skill building.

## How to Run

### Prerequisites

-   **Flex** installed on your system.
-   **GCC** compiler.

### Compilation and Execution

To compile and run any `.l` file in this project, follow these steps (using `exemplo.l` as an example):

1.  **Generate the C source code from the Flex file**:
    
    ```
    flex exemplo.l
    
    ```
    
2.  **Compile the generated C file**:
    
    ```
    gcc lex.yy.c -o main.exe
    
    ```
    
3.  **Run the executable**:
    
    ```
    ./main.exe < teste.txt
    
    ```
    
    _(Note: You can also run the program and type directly into the terminal, pressing `Ctrl+D` (Unix) or `Ctrl+Z` (Windows) to end the input.)_

## Project Structure

-   **Root Directory**: Contains standalone practice scripts like `simbol_counter.l`, `num_linhas.l`, and `nome_var.l`.
-   **`lista/`**: A subdirectory containing specific exercises (`ex1.l` to `ex6.l`) and their test files.
-   **`teste.txt` / `rascunho.txt`**: Sample text files used to verify the behavior of the lexical analyzers.
-   **`lex.yy.c`**: The intermediate C code generated by Flex (included for reference but typically ignored by version control).

## What I Learned

-   **Regular Expressions**: Designing complex patterns to match specific tokens in a stream of text.
-   **Flex Actions**: Associating C code actions with matched patterns to perform counts, transformations, or error logging.
-   **Input Streams**: Handling file redirection and standard input within the Flex environment.
-   **Scanner States**: Understanding how the generated finite automata transition between states to process text.
